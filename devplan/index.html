Domer Doors Dev Plan

Relational Schema of Database

Resident(netID varchar(8) primary key, year int not null, name varchar(50) not null, password varchar(32), roomNum varchar(5), dorm varchar(20), isStaff bool, party bool, study bool, closeToStaff bool, closeToStairs bool, closeToElevator bool, goodView bool, prefSection varchar(5), prefFloor int, foreign key (pickNum) references Pick(pickNum))

Section(dorm varchar(20), sectionID varchar(10), primary key(dorm, sectionID))

Room(dorm varchar(20), roomNum int, sectionID varchar(10) not null, numResidents int not null, floor int, hasGoodView bool, isOccupied bool, freshmanRoom bool, staffRoom bool, lounge bool, studyRoom bool, bathroom bool, sqareFootage int, closeToStairs bool, closeToElevator bool, primary key (dorm, roomNum), foreign key (sectionID) references Section(sectionID))

Pick(pickNum int primary key, netID varchar(8) unique, dorm varchar(20) primary key, foreign key (roomNum) references Room(roomNum), foreign key (netID) references Resident(netID))

Database, Platforms, and Languages

The group will be using MySQL, php, HTML/CSS and JavaScript for the project. The version control used will be GitHub.

Data Gathering

The group will gather data from the university's architect's office and administration office. The data will include floor plans of each dorm, designs of each building, and student netIDs. If no student netIDs are given, the group will ask for voluntary IDs of the students in our halls.

Who's Doing What

In terms of who will be doing what, the split will be even in the beginning. Each group member will do every task assigned to a dorm in terms of simple queries, basic html, php, etc., so every group member can be comfortable dealing with all aspects of web dev.

  • Ben - Dillion Hall
  • Cory - Fisher Hall
  • Jack - Carroll Hall
  • David - Knott Hall

As the project progresses, the split will still be relatively even, however, Jack and David will attempt to help the group implement more advanced UI features. Ben and Cory will be doing more advanced, in-depth query searching. Saying this, all will still be equally involved but with some more advanced features being handled by different members of the group based on experience in a certain field.
